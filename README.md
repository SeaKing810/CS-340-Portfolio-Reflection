Maintainable, Readable, and Adaptable Code

Writing maintainable, readable, and adaptable code starts with clear structure and separation of responsibilities. In Project One, I created a CRUD Python module that handled all database interactions independently from the rest of the application. This made the code easier to understand, since all database logic was centralized in one place rather than scattered throughout the project.

One of the biggest advantages of this approach was reusability. When I moved into Project Two and built the dashboard, I was able to reuse the same CRUD module without rewriting database logic. This made development faster and reduced the chance of errors. It also made debugging easier, since any issues related to the database could be traced back to a single file.

In the future, this CRUD module could be reused in other applications that require database interaction, such as web apps, APIs, or data dashboards. It provides a strong foundation for scalable and maintainable systems.

Problem-Solving Approach as a Computer Scientist

When approaching this project, I focused on breaking the problem down into smaller parts. First, I made sure the database was correctly set up and populated. Then, I developed the CRUD functionality to interact with the data. Finally, I built the dashboard to visualize and present the data in a meaningful way.

This approach was different from earlier assignments because it required integrating multiple components instead of working on isolated tasks. I had to think more about how each part of the system connected, rather than just solving a single problem.

In the future, I would continue using this structured approach by first understanding the requirements, then designing the system architecture, and finally implementing each component step by step. I would also spend more time planning before coding to ensure efficiency and scalability.

The Role of Computer Scientists

Computer scientists design and build systems that allow organizations to store, manage, and analyze data efficiently. Their work is important because it enables better decision-making, automation, and overall productivity.

In this project, the dashboard and database system would help a company like Grazioso Salvare quickly access and analyze data related to animal rescue operations. Instead of manually searching through data, they can use the dashboard to filter, visualize, and interpret information in real time.

This type of solution improves efficiency, reduces errors, and allows the company to focus more on their mission rather than administrative tasks. Overall, it demonstrates how technology can directly support and enhance real-world operations.
